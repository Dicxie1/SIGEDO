# Nomenclatura de ramas
las ramas debe de seguir un orden logico es por ello que se definir la siguiente nomenclatura para hacer contribución al proyecto.

## Formato de rama 

```
<prefijo>/<descripcion>
```
| prefijo | Descripcion | ejemplo |
|---------|-------------|---------|
| feat/   | Nuevas funcionalidades o mejoras. | feat/agregar-carrito-compras |
|fix/ | Corrección de errores o bugs | fix/error-login-facebook | 
| hotfix/ | Correcciones críticas en producción. | hotfix/caida-pagos-urgente |
| release/ | Preparación de una nueva versión. | release/v1.2.0 |
| docs/ |  Cambios en documentación. |  docs/actualizar-readme | 
| refactor/ | Mejoras en el código sin cambiar funcionalidad. | refactor/optimizar-consultas-sql |
| test/ | Añadir o modificar pruebas. | test/cobertura-login |
| chore/ | Tareas de mantenimiento (configuración, dependencias, etc.). | chore/actualizar-dependencias |

# Reglas 
- **Longitud**: Máximo de 50 caracter
- **caracteristicas**: Solo mínusculas, númerosy guiones(-)
